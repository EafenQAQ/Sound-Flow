# Proposal: Redesign Authentication UI

## Summary
重新设计登录和注册页面的用户界面，采用现代化的分屏布局设计，提升用户体验和视觉吸引力。

## Motivation
当前的登录和注册页面采用简单的居中表单布局，缺乏视觉层次感和品牌特色。通过引入分屏设计，可以在展示品牌形象的同时提供更专业的用户体验。

## Goals
1. 创建悬浮的长方形UI容器
2. 登录页面：左侧显示logo和"欢迎！"文字，右侧显示登录表单
3. 注册页面：左侧显示注册表单，右侧显示logo和"欢迎！"文字
4. 保持响应式设计，确保移动端体验
5. 保持现有的表单功能和交互逻辑不变
6. "欢迎！"文字使用匹配logo大小的字体，并添加出现动画

## Non-Goals
- 不修改认证逻辑或后端集成
- 不改变表单验证规则
- 不修改路由配置
- 不添加新的认证方式（如社交登录）

## Success Criteria
- 登录页面左侧显示logo和"欢迎！"文字，右侧显示表单
- 注册页面左侧显示表单，右侧显示logo和"欢迎！"文字
- UI容器具有悬浮效果（阴影、圆角）
- 在桌面端（>768px）显示分屏布局
- 在移动端（≤768px）垂直堆叠布局
- 所有现有功能正常工作（表单提交、错误显示、加载状态）
- "欢迎！"文字使用匹配logo大小的字体，并具有出现动画

## Risks
- Logo图片可能需要调整尺寸以适应新布局
- 虚化效果可能影响性能
- 移动端布局可能需要额外优化

## Dependencies
- 无外部依赖
- 需要确认 logo.webp 文件存在且可访问

## Related Changes
- 无相关变更
