# 变更：添加全屏播放器覆盖层组件

## 为什么
当前的音乐播放器是一个固定在底部的迷你播放栏，提供基本的播放控制功能。用户希望获得更沉浸式的音乐体验，包括更大的专辑封面、更易读的歌曲信息和增强的控制功能。全屏覆盖层将提供一个专注、无干扰的音乐播放模式，特别是在屏幕空间有限的移动设备上更加实用。

## 变更内容

### 桌面端
- 添加一个新的全屏播放器覆盖层组件，可以从现有的底部播放栏切换，向上滑动动画进入全屏组件
- 左半边显示旋转的唱片（占据 50% 宽度）
- 右半边显示当前歌单的歌曲列表（占据 50% 宽度），可点击切换歌曲，当前歌曲高亮
- 顶部显示歌曲标题、歌单名和艺术家名称
- 底部显示迷你播放器控件
- 左上角添加"下箭头"按钮以退出全屏模式，向下滑动的动画

### 移动端
- 采用垂直堆叠布局，参考网易云音乐 App 的设计
- 唱片占据屏幕宽度的 80% 并居中显示，播放时旋转
- 唱片下方显示歌曲标题和艺术家名称
- 进度条显示在歌曲信息下方
- 播放控制按钮（上一首、播放/暂停、下一首）显示在底部，不显示音量控件
- 左上角添加"下箭头"按钮以退出全屏模式，向下滑动的动画

### 通用特性
- 全屏播放器整个背景是歌单的封面图，做微微的虚化（blur(10px)）和透明（opacity: 0.8）效果
- 屏幕整体有毛玻璃效果（backdrop-filter: blur(20px)）
- 迷你播放器控件透明度变低（opacity: 0.6 blur(20px)），与全屏控件整体统一
- 透明度和模糊效果的变化要有平滑的动画
- 支持桌面端和移动端的响应式布局
- 在迷你播放器和全屏模式之间切换时保持播放状态
- 添加进入/退出全屏模式的平滑过渡动画——上下滑动的动画（300ms）
- 点击迷你播放控件的任何地方（除了那几个功能按钮之外），都可以唤出全屏组件
- 全屏组件的所有播放控件用已有逻辑，不需要额外添加逻辑，只是额外做一个全屏组件 UI

## 影响范围
- 受影响的规范：`music-player`（新能力）
- 受影响的代码：
  - `src/components/global/MusicPlayer.vue` - 添加切换按钮和覆盖层组件
  - `src/stores/player.js` - 添加全屏模式状态
  - `src/main.css` - 添加覆盖层样式和动画
